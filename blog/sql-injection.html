SQL Injection
Oct. 12

I used to be amazed by hackers. We can constantly hear some news about a large firm was hacked and hundreds of thousands of its customers' personal data are stolen. It seems like they are a step ahead of the company's developer. It would be great if they can use the skill for good.

Turns out that's not quite all about skills.

Let's talk about SQL injection. So most of the data of a company will be stored in the form of database, using SQL statements to access the data.

For example, this company has a website that allows user to use their username and password to log in to their account. Then there is this two input block for you to put in your username and password. In most cases, after you input your information in these two blocks, the website will use these content as part of the SQL command to bring up your account from the database. Bascially, the inputs will be added into a query, then the database server will use this query to match your username and password using the simple equal statement to bring up your account. If either one of these two pieces of information does not match, then the database server will not return much information other than "Your username or password is incorrect".
